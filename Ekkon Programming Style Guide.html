Ekkon Programming Style Guide

Note that this style guide is not a complete reference. It builds upon ideas that are seen as common practice, listing only the styles that Ekkon has changed. Refer to the Google C++ style guide for any missing rules. Questions may be forwarded to bobbyewy@gmail.com.

Documentation

Every header file must have its own comment header at the top. This must be above 
#include
s and other compiler directives. A header should be layed out as follows: 
/*
 * Class: Rectangle
 * Author: John Doe
 * Date: 01/05/2016
 * Description: A simple header-only class for rectangles. More
                sample text to demonstrate newlines.
 */
Dates must be in the form 
DD/MM/YYYY
. Headers must be added to files at the time of creation to ensure completeness of the comment header.

Files and File Structure

Every source file must have one header file to match. Header files must contain one and only one class. Source files may not declare classes; they must implement class functions as follows: 
void Rectangle:operate() {

}
Every header file must have the 
#pragma once
directive immediately below its comment header, only preceded by a single blank line. Do not use header guards! They are an old convention left over from compilers that did not provide similar directives. 
/*
 * ...
 */

#pragma once

// Do not do this.
#ifndef CLASSNAME_HEADER_GUARD
#define CLASSNAME_HEADER_GUARD
...
#endif

Functions

Always indent function bodies properly. Braces should be on the same line as the function. 
void operate() {
}

// Do not do this.
void operate()
{
}
If a function has no parameters, do not specify void. 
// Do not do this.
void operate(void);
When declaring funtions in header files, name the parameters. 
// Do not do this.
void operate(int, float);

Pointers and References

Pointers must be declared as follows: 
int* pointer1;
Pointer dereferences must have no space between the * and variable name. Obtaining a pointer must also be done in similar fashion. 
int value1 = *pointer1;
int* pointer2 = &value1;
References must be formatted in the same way. 
int& value2 = value1;
